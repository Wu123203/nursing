### 2.2 前端验证码功能修复（已完成 ✅）

**问题**：登录页面验证码图片无法正常显示

**原因**：前后端响应格式不匹配
- 后端返回：`{code: 200, data: {image: "base64...", captchaKey: "xxx"}}`
- 前端原来直接使用 `res.data` 作为图片数据

**修复**：
- 修改 `Login.vue`，正确解析 `res.data.image` 和 `res.data.captchaKey`
- 图片格式改为 `data:image/gif;base64,`

### 2.3 登录错误信息显示修复（已完成 ✅）

**问题**：某些错误场景下登录错误信息无法正常显示

**原因**：
1. HTTP 错误状态码（400/401/500）时，前端拦截器直接拒绝 Promise
2. 错误消息字段不一致（`msg` vs `message` vs `error`）

**修复**：
- 增强 `request.js` 错误处理，提取响应数据传递给业务逻辑
- `Login.vue` 支持多种错误消息字段提取
- 登录失败后自动刷新验证码

## 四、Redis Key 设计规范

### 4.1 Key 常量定义（RedisConstants.java）
```java
// 验证码
public static final String CAPTCHA_KEY = "captcha:";
public static final long CAPTCHA_TTL = 300; // 5分钟

// 登录失败次数
public static final String LOGIN_FAIL_KEY = "login:fail:";
public static final int MAX_LOGIN_FAIL = 5;

// 统计数据缓存
public static final String COUNT_CACHE_KEY = "cache:count";
public static final long COUNT_CACHE_TTL = 300; // 5分钟

// Token 黑名单
public static final String TOKEN_BLACKLIST_KEY = "token:blacklist:";

// 新闻浏览量
public static final String VIEW_COUNT_KEY = "view:count:news:";

// 点赞数
public static final String LIKE_COUNT_KEY = "like:count:";
public static final String LIKE_USER_KEY = "like:user:";

// 限流
public static final String RATE_LIMIT_KEY = "rate:limit:";
```

### 4.2 Redis 数据结构
| Key 模式 | 数据类型 | 用途 |
|----------|----------|------|
| `captcha:{uuid}` | String | 验证码内容 |
| `login:fail:{username}` | String | 登录失败次数 |
| `cache:count` | String | 统计数据 JSON |
| `token:blacklist:{tokenHash}` | String | Token 黑名单 |
| `view:count:news:{id}` | String | 新闻浏览量 |
| `like:count:{newsId}` | String | 点赞数 |
| `like:user:{newsId}` | Set | 点赞用户集合 |
| `rate:limit:{ip}:{uri}` | String | 限流计数器 |

## 六、注意事项

### 6.1 环境依赖
1. **Redis 必须运行**：验证码、登录、缓存等功能依赖 Redis
2. **MySQL 必须运行**：数据库 `db_elderly_care` 需存在
3. **Java 17+**：Spring Boot 3.0 需要 Java 17+

### 6.2 配置文件
- 后端配置：`elderly-care-server/src/main/resources/application.yml`
- Redis 地址：`192.168.25.128:6379`（密码：`112233`）
- MySQL 地址：`localhost:3306`（用户：`root`，密码：`112233`）

### 6.3 定时任务
- 新闻浏览量同步：每 10 分钟执行
- 点赞数同步：每 10 分钟执行
- 需确保启动类有 `@EnableScheduling` 注解

### 6.4 安全配置
- 验证码接口：`/captcha/**` 已配置为公开访问
- 登录接口：`/login` 已配置为公开访问
- 其他接口需携带 JWT Token
